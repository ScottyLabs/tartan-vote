TartanVote
UI Redesign Proposal · v1
Visual only — no code changed
Concept

A calmer, more official-feeling voting experience — built for clarity under pressure.

The current UI keeps all the right entries and actions, but the red-on-red stacked buttons and flat modals make it feel unfinished. This proposal keeps every feature intact and re-skins the app with a considered type system, a governance-grade color palette, and layouts that emphasize what's happening right now.

Design principles
  • 1
    Stay official. CMU scarlet, but refined and used with restraint.
  • 2
    Reduce modal anxiety. Structured forms with hierarchy and previews.
  • 3
    Show live state. Status chips, pulses, and motion timers are first-class.
  • 4
    Same entries. Every field and control from the current app is preserved.

09 · Before / after

What changes, what stays

Every button, input, and flow you already have is preserved. The differences are visual and structural: hierarchy, spacing, color discipline, and contextual affordances.

Current
  • • All-red, all-uppercase buttons stack the same way
  • • Modals are tall, flat white boxes with minimal hierarchy
  • • No persistent navigation — host uses back-arrow only
  • • Results are a small centered card with thin bars
  • • Motion / election entries feel identical
  • • "X" close control lives in isolation, not paired with title
Proposed
  • • Primary / ghost / dark button roles used consistently
  • • Modals pair form + live preview; clear section groups
  • • Dark rail sidebar anchors the host and surfaces meeting code
  • • Results get an editorial headline + status bar on top
  • • Motions and elections have distinct headers and iconography
  • • Close sits next to title; status chips replace plain text
